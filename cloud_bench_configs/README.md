# Workload config (read from CBS)

**Do not maintain workload JSON in this repo.**

| Workload | Canonical CBS path |
|----------|-------------------|
| Online (8 services) | `colocation-bench-suite/cloud_bench_configs/workloads.cloud.json` |
| Offline catalog | `colocation-bench-suite/offline_bench_configs/workloads.offline.json` |
| Offline matrix CSV | generated by `colocation-bench-suite/scripts/offline_matrix.sh` |

IPT collectors default to `$CBS_ROOT/cloud_bench_configs/workloads.cloud.json`
(set `CBS_ROOT` or `COLOCATION_BENCH_SUITE_DIR`).

Startup is delegated to CBS:

- Online: `scripts/cloud_workload_lib.py`
- Offline: `scripts/offline_workload_lib.py`

Service tuning (threads, QPS, DB sizes) lives only in CBS
`conf/profiles/cloud_realistic.env`.

## Machine-specific overrides

Copy or symlink an overlay JSON **next to CBS**, not under IPT:

```bash
cp cloud_bench_configs/workloads.cloud.json \
   /path/to/overlays/workloads.machine.json
# edit cpusets / durations
```

Then pass to IPT:

```bash
python3 scripts/collect/run_cloud_perf_trace_analysis.py \
  --workload-config /path/to/overlays/workloads.machine.json
```

See `colocation-bench-suite/docs/CBS_IPT_ALIGNMENT.md` for the full split.
